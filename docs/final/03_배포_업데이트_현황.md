# 배포 및 업데이트 현황

## 1. 현재 결론
### 가. `NEW\source` 기준으로 배포용 파일 만들기
가능합니다. 새 표준 build pipeline으로 `v202603111526` 빌드를 실제 생성했습니다.

생성 결과:
- release 폴더: `NEW\build\dist\v202603111526\MonitoringDashboard`
- ZIP: `NEW\build\artifacts\v202603111526\MonitoringDashboard.zip`
- SHA256: `3b1da2acd4f019d59459b84ea3296937da439b9f2ac608c8e586e708ac2e7367`

현재 배포 방식:
- source -> staging 복사
- PyArmor 부분 암호화
- PyInstaller onedir 패키징
- ZIP, SHA256, manifest, release notes 생성

추가 검증:
- 빌드된 `MonitoringDashboard.exe` isolated launch 테스트 성공

### 나. GitHub에 업로드하기
업로드용 스크립트는 준비됐고, repo 정보도 manifest에 정상 반영되도록 수정했습니다.

준비된 파일:
- `NEW\source\publish_release.bat`
- `NEW\source\publish_release.ps1`

현재 상태:
- `github_repo = gyers1/monitoring-updates`
- `UPDATE_URL` 기준 repo 추출 가능
- 아직 실제 업로드는 하지 않음

### 다. 프로그램 설정란에서 업데이트 버튼으로 적용하기
구조상 가능하며, 새 배포 ZIP도 updater가 기대하는 구조로 생성됐습니다.

남은 검증:
1. GitHub latest release에 새 ZIP 업로드
2. 테스트 설치본에서 `설정 > 업데이트` 실행
3. 자동 교체 및 재시작 확인

## 2. 현재 암호화 기준
- 현재 PC의 PyArmor는 trial이라 전체 폴더 암호화는 중간에 실패함
- 그래서 현재 표준은 아래 3개만 부분 암호화
  - `main.py`
  - `webview_app.py`
  - `application/crawl_service.py`

이 방식은 실제 빌드 성공까지 확인됨
