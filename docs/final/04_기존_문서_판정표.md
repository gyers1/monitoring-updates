# 기존 문서 판정표

이 문서는 `NEW\docs` 안의 기존 파일을 하나씩 확인한 뒤, 현재 기준에서 어떤 용도로 남길지 정리한 기록입니다.

## 1. 파일별 판정
- `DEVELOPER_GUIDE.md`
  - 판정: 참고 유지
  - 이유: 영어 기준으로는 읽을 수 있고 기술 개요와 업데이트 구조 설명이 남아 있음. 다만 경로가 옛 기준이고 한글 경로 표기는 일부 깨져 있음.

- `FINAL_BACKUP_20260127.md`
  - 판정: 참고 유지
  - 이유: 과거 정상 백업 시점 기록으로 유용함. 현재 운영 기준 문서는 아님.

- `개발명세서.md`
  - 판정: 대체 필요
  - 이유: 본문 한글이 많이 깨져 있고 경로 기준도 옛 구조임. 현재 기준은 `docs\final` 문서로 대체.

- `국회형_사이트_템플릿_추가_절차.md`
  - 판정: 참고 유지
  - 이유: 사이트 유형 확장 시 참고 가치가 있음. 다만 인코딩이 깨져 있어 재작성 전까지는 참고용만 가능.

- `배포_빌드_암호화_가이드.md`
  - 판정: 참고 유지
  - 이유: PyArmor/PyInstaller 절차의 뼈대는 남아 있음. 하지만 경로와 출력 규칙이 `NEW` 기준이 아니므로 그대로 따르면 안 됨.

- `배포_업데이트_가이드.md`
  - 판정: 역사 자료
  - 이유: 배포와 업데이트 흐름의 의도는 파악 가능함. 하지만 `release_source_webview`, `release_dist_webview_*` 등 옛 경로 기준이라 현재 표준 문서로 쓰기 부적절.

- `비개발자_유지보수_가이드.md`
  - 판정: 대체 필요
  - 이유: 역할 설명은 있으나 본문 인코딩이 많이 깨짐. 현재 기준 유지보수 설명은 새 문서로 대체.

- `선택자_링크_해석_정책.md`
  - 판정: 참고 유지
  - 이유: 링크 해석 정책 자체는 중요함. 다만 문서 품질이 좋지 않아 필요 시 UTF-8로 다시 정리해야 함.

- `업데이트_정상화_원인_요약_20260210.md`
  - 판정: 참고 유지
  - 이유: 비교적 읽을 수 있고, 업데이트 정상화 원인과 운영 체크리스트가 명확함.

- `오류_수정_노트.md`
  - 판정: 참고 유지
  - 이유: 과거 장애 원인과 수정 내역이 누적돼 있어 가치가 있음. 다만 일부 구간 한글이 깨져 있음.

- `참고자료\ui_prefs_from_v8.json`
  - 판정: 참고 자료
  - 이유: 데이터 샘플로는 의미가 있으나 운영 문서는 아님.

- `참고자료\국회.md`
  - 판정: 참고 자료
  - 이유: 원시 페이지 보관본에 가까움. 문서라기보다 분석용 캡처 데이터임.

## 2. 최종 판단
- 현재 기준 문서는 `NEW\docs\final`만 본다.
- 기존 `NEW\docs`는 역사 자료와 참고 자료로 남긴다.
- 깨진 문서를 억지로 이어 쓰지 말고, 필요한 주제만 새 UTF-8 문서로 계속 대체한다.
