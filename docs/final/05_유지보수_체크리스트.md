# 유지보수 체크리스트

## 1. 가장 우선할 일
- 새 빌드 `v202603111526`를 GitHub Release로 올릴지 결정
- 테스트 설치본에서 앱 내 업데이트 1회 검증
- 필요하면 `NEW\release`를 새 빌드 기준으로 갱신

## 2. 릴리즈 전 체크리스트
- 버전 태그를 새 값으로 올렸는지 확인
- `build_release.bat -Version vYYYYMMDDHHMM`로 새 빌드 생성
- ZIP과 SHA256 기록 확인
- manifest의 `github_repo`, `update_url` 값 확인
- 테스트 폴더에서 실행 확인
- 업로드 후 업데이트 버튼 검증

## 3. 운영 부채
- PyArmor 정식 라이선스가 없어서 전체 폴더 암호화는 아직 불가
- GitHub 업로드와 업데이트는 아직 실제 end-to-end 검증 전
- 기존 `NEW\archive` 용량은 여전히 큼
- 민감 정보(.env, 토큰) 분리 정책은 추가 정리 필요

## 4. 권장 운영 방식
- 개발: `NEW\source`
- 실행 확인: `NEW\release`
- 새 산출물: `NEW\build`
- 문서: `NEW\docs\final`
- 과거 결과물: `NEW\archive`
