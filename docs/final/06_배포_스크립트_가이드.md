# 배포 스크립트 가이드

## 1. 새 기준 흐름
1. `setup_build_env.bat`
2. `build_release.bat -Version vYYYYMMDDHHMM`
3. 생성된 ZIP, SHA256, manifest 확인
4. 테스트 폴더에서 실행 확인
5. `publish_release.bat -Version vYYYYMMDDHHMM`
6. 테스트 설치본에서 업데이트 버튼 검증

## 2. 사용 파일
- `setup_build_env.bat`: Python 3.12 venv 및 build 도구 준비
- `build_release.bat`: 암호화 포함 배포 빌드 생성
- `publish_release.bat`: GitHub Release 업로드
- `deploy_assets\start_tray.bat`: 배포본에 복사되는 공식 런처 템플릿

## 5. 현재 암호화 방식
현재는 아래 3개 파일만 PyArmor 부분 암호화합니다.
- `main.py`
- `webview_app.py`
- `application/crawl_service.py`

이유:
- 현재 PyArmor trial 한계 내에서 실제 빌드가 성공하는 방식
- 핵심 진입점과 크롤링 서비스 로직 보호 가능
- 템플릿/정적 파일 손상 위험 감소

## 6. 이전 스크립트 처리
- `_build_pyinstaller.cmd`는 더 이상 직접 유지하지 않음
- 현재는 `build_release.bat`로 연결되는 안내용 파일로 사용
